        ↓

        ↓

        ↓

        ↓

JWT Authentication
user logs in->server verifies credentials->generates jwt->client stores jwt-> token sent in future requests
enables -> protected routes, role-based access, secure api
Jwt - signed token proves identity, contains userid role and expiration

Auth middleware - protect routes
middleware - verifies jwt token, identifies loggedin user, protects private routes
so that only loggedin users can access apis, identify who is making request, for role based access
Middleware  = a function that runs request->middleware->route->response. sits between request and response
client send req with token->middleware checks token->if valid - continue, else reject


Authentication - Who are you? - jwt loggedin
Authorization = What can you do? - roles based

student selects food -> places order-> order stored in db->cook sees order-> cook updates status-> student tracks order
states = Pending->Accepted->Preparing->Delivered

Order system
connects student->food->cooks
student places order -> order stored in db-> cook views incoming orders->cook updates order status -> student tracks order

compute price in backend
owner is cook, no other order can modify the status
populate() - used to fetch related data

API -> POST   /api/orders              → place order
GET    /api/orders/my-orders    → student orders
GET    /api/orders/cook-orders  → cook orders
PUT    /api/orders/:id/status   → update status
